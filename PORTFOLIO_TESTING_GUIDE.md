## 🧪 Пошаговое тестирование

### Шаг 1: Запуск dev сервера

```bash
npm run dev
```

Ожидается:

- ✅ Нет ошибок компиляции
- ✅ Server запустился на localhost:3000
- ✅ Sanity client инициализирован

### Шаг 2: Открить Portfolio страницу

```
http://localhost:3000/en/portfolio
```

Проверка:

- ✅ Страница загружается без ошибок
- ✅ Заголовок "My Portfolio" видно
- ✅ Подзаголовок отображается (если заполнен)

### Шаг 3: Проверить документы

Если документы есть в Sanity:

- ✅ Они отображаются в виде карточек
- ✅ Иконки файлов видны (📄 для PDF, 📘 для Word)
- ✅ Названия документов видны
- ✅ Описания видны (если заполнены)
- ✅ Метаинформация видна (тип, имя файла)

Если документов нет:

- ✅ Отображается сообщение "No documents available"

### Шаг 4: Протестировать Preview (PDF)

1. Нажмите кнопку **[Preview]** для PDF документа
2. Проверка:
   - ✅ PDF открывается в новой вкладке
   - ✅ Документ отображается корректно
   - ✅ Инструменты просмотра работают

### Шаг 5: Протестировать Download

1. Нажмите кнопку **[Download]** для любого документа
2. Проверка:
   - ✅ Файл скачивается
   - ✅ Имя файла правильное
   - ✅ Файл открывается корректно

### Шаг 6: Протестировать локализацию

Переключитесь на другие языки:

```
http://localhost:3000/uk/portfolio
http://localhost:3000/ru/portfolio
```

Проверка:

- ✅ Текст переводится правильно
- ✅ Все элементы видны на каждом языке
- ✅ Функциональность сохраняется

### Шаг 7: Адаптивное тестирование

Откройте DevTools (F12) и используйте Device Emulation:

#### Mobile (375px width)

- ✅ Карточки выстраиваются в один столбец
- ✅ Кнопки расположены вертикально
- ✅ Текст читаемый
- ✅ Нет горизонтального скролла
- ✅ Иконки видны

#### Tablet (768px width)

- ✅ Карточки в один столбец
- ✅ Элементы хорошо выглядят
- ✅ Удобно для навигации

#### Desktop (1024px+ width)

- ✅ Карточки в один столбец (full width)
- ✅ Элементы правильно выровнены
- ✅ Hover эффекты работают

### Шаг 8: Проверить консоль

Откройте DevTools → Console

- ✅ Нет красных ошибок
- ✅ Нет желтых предупреждений (кроме external)
- ✅ Debug logs только если нужны

### Шаг 9: Проверить производительность

DevTools → Performance:

```bash
1. Откройте DevTools
2. Перейдите на Portfolio страницу
3. Нажмите Record
4. Скроллите вниз и взаимодействуйте
5. Нажмите Stop
```

Проверка:

- ✅ FCP < 1500ms
- ✅ LCP < 2500ms
- ✅ CLS < 0.1
- ✅ Smooth scrolling
