# Normative Base - Quick Start Guide 🚀

## За 5 минут до первого документа!

### Шаг 1: Откройте Sanity Studio

```
npm run dev:sanity
```

Или перейдите на `http://localhost:3333/`

### Шаг 2: Создайте Normative Base документ

1. В левой панели кликните **"Create"**
2. Выберите тип **"Normative Base"**
3. Заполните основные поля:
   - **Section Title**: "Normative Base" (или ваше название)
   - **Section Subtitle**: "Access regulatory documents" (или описание)

### Шаг 3: Добавьте первый документ

1. Прокрутите вниз до раздела **"Documents"**
2. Кликните кнопку **"Add"**
3. Заполните поля:
   - **Document Title**: "Charter" (название документа)
   - **Document Description**: "Main regulations..." (опциональное описание)
   - **Document File**: Кликните и загрузите PDF или Word файл

### Шаг 4: Опубликуйте документ

1. Кликните кнопку **"Publish"** в правом верхнем углу
2. Подождите пока Sanity сохранит изменения

### Шаг 5: Посмотрите на сайте

1. Откройте браузер: `http://localhost:3000/en/normative-base`
2. Вы должны увидеть ваш документ!

## 📱 Вид на разных языках

- **English**: http://localhost:3000/en/normative-base
- **Русский**: http://localhost:3000/ru/normative-base
- **Українська**: http://localhost:3000/uk/normative-base

## ⚙️ Редактирование существующих документов

1. В Sanity найдите существующий документ "Normative Base"
2. Кликните на документ в списке
3. Отредактируйте нужные поля
4. Кликните "Publish" (или "Unpublish Deploy" если нужно сохранить черновик)

## ❌ Удаление документа

1. Откройте Normative Base документ в Sanity
2. В списке Documents наведите на документ
3. Кликните иконку **удаления** (корзина)
4. Кликните "Publish" для применения изменений

## 📝 Добавление описания к документу

1. При добавлении/редактировании документа заполните поле **"Document Description"**
2. Описание будет отображаться серым текстом под названием
3. Это поле опциональное - пропустите его, если не нужно

## 🎨 Многоязычность

Каждое поле можно редактировать на разных языках:

1. Кликните на язык в правом верхнем углу (en/ru/uk)
2. Отредактируйте текст на этом языке
3. Повторите для всех языков

**Пример:**

- **English**: Document Title = "School Charter"
- **Russian**: Document Title = "Устав школы"
- **Ukrainian**: Document Title = "Статут школи"

## 📊 Лучшие практики

### ✅ ДА:

- Используйте понятные названия документов
- Добавляйте описания для важных документов
- Группируйте похожие документы по типам
- Регулярно обновляйте устаревшие документы
- Проверяйте файлы перед загрузкой

### ❌ НЕТ:

- Не загружайте файлы > 10 MB (медленная загрузка)
- Не оставляйте пустые поля (если они обязательные)
- Не добавляйте слишком похожие документы
- Не используйте спецсимволы в названиях

## 🔄 Обновление документа

1. Откройте документ в Sanity
2. Отредактируйте содержимое (замените файл, название и т.д.)
3. Кликните "Publish" (изменения появятся сразу на сайте)

## 🆘 Часто задаваемые вопросы

### Q: Как изменить заголовок страницы?

**A:** Отредактируйте поле "Section Title" в документе Normative Base.

### Q: Документ не отображается на сайте

**A:** Убедитесь, что:

1. Документ опубликован (кнопка "Publish" кликнута)
2. На странице обновлена (F5 или Ctrl+Shift+R)
3. Вы на правильном языке

### Q: Можно ли переупорядочить документы?

**A:** Да! Перетащите документы в нужном порядке в списке Documents.

### Q: Какой максимальный размер файла?

**A:** До 2 GB, но рекомендуется < 10 MB для быстрой загрузки.

### Q: Как добавить еще одну страницу нормативных документов?

**A:** Создайте новый документ типа "Normative Base" (можете иметь несколько).
