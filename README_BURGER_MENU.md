## 🎯 Что было сделано

### 1️⃣ Санity Схема ✅

**Файл**: `sanity/schemaTypes/menuNavigation.js`

Создана полная схема для управления меню:

- Document type: `menuNavigation`
- Поле: `menuItems[]` - массив пунктов
- Каждый пункт содержит:
  - `title` - название (en, ru, uk)
  - `slug` - путь в URL
  - `order` - порядок сортировки
  - `children` - вложенные пункты (опционально)

**Использование в Sanity Studio**:

- Администраторы редактируют в `/studio`
- Добавляют пункты меню как документ
- Заполняют переводы на все языки
- Нажимают "Publish"

### 2️⃣ GROQ Запрос и Преобразование ✅

**Файл**: `sanity/lib/queries.js`

Добавлены три функции:

```javascript
// 1. GROQ запрос для получения меню
export const MENU_QUERY = `*[_type == "menuNavigation"][0]{ ... }`

// 2. Функция загрузки из Sanity
export async function getMenuData() { ... }

// 3. Функция преобразования в формат компонента
export function mapMenuData(data, locale) { ... }
```

**Что делают**:

- Загружают сырые данные из Sanity
- Преобразуют в удобный формат для компонента
- Достают локализованные значения (title на нужном языке)

### 3️⃣ API Endpoint ✅

**Файл**: `app/api/menu/route.js`

Создан API endpoint для фронтенда:

- **URL**: `GET /api/menu?locale=en`
- **Параметры**: `locale` (en, ru, uk)
- **Ответ**: JSON с массивом `menuItems`
- **Обработка ошибок**: Try-catch, graceful fallback

**Как работает**:

1. Frontend делает запрос к `/api/menu?locale=en`
2. API вызывает `getMenuData()` из Sanity
3. API преобразует данные через `mapMenuData()`
4. Возвращает JSON фронтенду

### 4️⃣ Обновленный BurgerMenu Компонент ✅

**Файл**: `app/components/BurgerMenu/BurgerMenu.js`

Полностью переработан компонент:

**Было**:

```javascript
const menuItems = [
  /* статический массив */
]
```

**Стало**:

```javascript
const [menuItems, setMenuItems] = useState([])
const locale = useLocale()

useEffect(() => {
  fetch(`/api/menu?locale=${locale}`)
    .then((res) => res.json())
    .then((data) => setMenuItems(data.menuItems))
}, [locale])
```

**Логика отображения**:

- Если `item.children.length > 0` → отображает Accordion
- Иначе → отображает обычную Link

```

    ↓
API endpoint (/api/menu/route.js)
    ↓ Вызывает
    ↓
getMenuData() → GROQ запрос → Sanity
    ↓ Вызывает
    ↓
mapMenuData(data, locale)
    ↓ Преобразует в формат компонента
    ↓
Возвращает JSON в компонент
    ↓
BurgerMenu отображает меню
    ↓
ПОЛЬЗОВАТЕЛЬ видит Burger Menu
```

## 📊 Функциональность

### ✅ Функция 1: Условное отображение

```javascript
if (item.children?.length > 0) {
  // Показать Accordion
  ;<Accordion>
    <div>Sub-item 1</div>
    <div>Sub-item 2</div>
  </Accordion>
} else {
  // Показать обычную ссылку
  ;<Link href={item.href}>{item.label}</Link>
}
```

### ✅ Функция 2: Многоязычность

- Пункты меню заполняются на трёх языках
- `mapMenuData()` берет текст на нужном языке
- При смене языка меню обновляется

### ✅ Функция 3: Сортировка

- Поле `order` в Sanity
- `mapMenuData()` сортирует по `order`
- Администратор может менять порядок без кода

### ✅ Функция 4: Динамические ссылки

- `slug` преобразуется в `href` (e.g., "about" → "/about")
- Поддерживаются вложенные пути (e.g., "graduates/2024")

### ✅ Функция 5: Управление через CMS

- Полностью управляется в Sanity Studio
- Админ может добавлять/удалять/редактировать без кода
- Публикация в real-time (или с кешированием)


### Шаг 2: Запуск проекта

```bash
npm run dev
# Проект должен запуститься без ошибок
```

### Шаг 3: Создание меню в Sanity (администратор)

1. Откройте `http://localhost:3000/studio` (или production `/studio`)
2. Нажмите "Navigation Menu"
3. Нажмите "Create" или "Publish"
4. Добавьте первый пункт меню:
   - Title (en): Home
   - Slug: home
   - Order: 1
5. Нажмите "Publish"

### Шаг 4: Проверка на сайте

1. Откройте главную страницу сайта
2. Нажмите на иконку Burger Menu (☰)
3. Должны увидеть "Home" в меню
4. ✅ Готово!
