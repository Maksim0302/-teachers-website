# Начальная настройка Burger Menu в Sanity CMS

## 🚀 Первые шаги после развертывания

### Шаг 1: Откройте Sanity Studio

1. Перейдите на `http://localhost:3000/studio` (или ваш production URL `/studio`)
2. Вы должны увидеть "Navigation Menu" в левой боковой панели

### Шаг 2: Создайте первый документ Menu

Если это первый раз, вам может быть предложено создать документ:

1. Нажмите на "Navigation Menu"
2. Нажмите кнопку "Create" или "New"
3. Начинайте добавлять пункты меню

### Шаг 3: Добавьте первый пункт меню

**Пример: простой пункт "Home"**

```
Title (English): Home
Title (Russian): Главная
Title (Ukrainian): Головна

Slug: home
Order: 1
Sub-menu Items: (оставьте пустым)
```

Нажмите "Add Menu Items" и заполните поля:

| Поле                       | Значение |
| -------------------------- | -------- |
| **Title** → en (English)   | Home     |
| **Title** → ru (Russian)   | Главная  |
| **Title** → uk (Ukrainian) | Головна  |
| **Slug**                   | home     |
| **Order**                  | 1        |

✅ Нажмите "Publish" для сохранения

### Шаг 4: Добавьте пункт с Accordion

**Пример: "About" с подпунктами**

```
Title (English): About
Title (Russian): О нас
Title (Ukrainian): Про нас

Slug: about-us
Order: 2

Sub-menu Items:
  1. Title (en): About Us, Slug: about, Order: 1
  2. Title (en): Our Team, Slug: team, Order: 2
  3. Title (en): History, Slug: history, Order: 3
```

Пошагово:

1. Нажмите "Add Menu Items"
2. Заполните Title и Slug
3. **Важно**: В поле "Sub-menu Items" нажмите "Add"
4. Добавьте первый подпункт:
   - Title (en): About Us
   - Slug: about
   - Order: 1
5. Повторите для остальных подпунктов
6. ✅ Нажмите "Publish"

### Шаг 5: Проверьте результат

1. Откройте главную страницу сайта
2. Нажмите на иконку Burger Menu (три линии ☰)
3. Вы должны увидеть:
   - "Home" как простую ссылку
   - "About" как Accordion (нажмите, чтобы раскрыть подпункты)

## 🎯 Практический пример пошагово

### Добавление "About" с подпунктами

#### Шаг 1: Откройте форму добавления

- Нажмите "Add Menu Items"
- Появится форма для заполнения

#### Шаг 2: Заполните основные поля

- **Title** вкладка "English": введите "About"
- **Title** вкладка "Russian": введите "О нас"
- **Title** вкладка "Ukrainian": введите "Про нас"
- **Slug**: введите "about-us"
- **Order**: введите "2"

#### Шаг 3: Добавьте подпункты (Sub-menu Items)

- Нажмите кнопку "Add" в разделе "Sub-menu Items"
- Появится форма для первого подпункта

##### Подпункт 1: About Us

- **Title** → en: "About Us"
- **Slug**: "about"
- **Order**: "1"

##### Подпункт 2: Our Team

- Нажмите "Add" еще раз для второго подпункта
- **Title** → en: "Our Team"
- **Slug**: "team"
- **Order**: "2"

##### Подпункт 3: History

- Нажмите "Add" еще раз для третьего подпункта
- **Title** → en: "History"
- **Slug**: "history"
- **Order**: "3"

#### Шаг 4: Сохраните

- Нажмите кнопку "Publish" внизу справа

## ✨ После публикации

### На сайте появится:

1. **В Burger Menu (☰)**:
   - Home (обычная ссылка)
   - About (Accordion с раскрывающимся списком)
     - About Us → ссылка на `/about`
     - Our Team → ссылка на `/team`
     - History → ссылка на `/history`
   - И остальные пункты...

### Автоматически:

✅ Меню обновляется на всех языках
✅ Порядок соответствует значению "Order"
✅ Accordion работает корректно
✅ Все ссылки работают

## 🔗 Правила для Slug

| ✅ Правильно     | ❌ Неправильно   | Почему                                |
| ---------------- | ---------------- | ------------------------------------- |
| `home`           | `/home`          | Не нужен leading slash                |
| `about-us`       | `About Us`       | Используйте lowercase и дефисы        |
| `team`           | `team-page`      | Хранится как есть в URL               |
| `graduates/2024` | `graduates-2024` | Можно использовать слеш для структуры |
| (пусто)          | `groups-only`    | Пусто, если только для группировки    |

1. Откройте сайт на мобильном устройстве
2. Нажмите на иконку Burger Menu (☰)
3. Проверьте:
   - ✅ Меню открывается
   - ✅ Accordion раскрывается при клике
   - ✅ Ссылки работают
   - ✅ Меню закрывается при клике на ссылку
   - ✅ Анимации плавные

## 🌍 Поддержка языков

При переключении языка (LangSwitcher):

1. **Нажмите на EN / РУ / УК** в правом углу
2. **Burger Menu должен обновиться на новый язык**
3. Все заголовки пунктов должны быть переведены

Если язык не меняется:

- ✓ Убедитесь, что вы заполнили все языки в Title
- ✓ Проверьте, что язык код правильный (en, ru, uk)

## 🐛 Диагностика проблем

### Проблема: Меню не отображается

```
☑ Решение:
1. Откройте /studio → Navigation Menu
2. Проверьте, что документ создан (должна быть кнопка "Publish")
3. Проверьте, что есть хотя бы 1 пункт в Menu Items
4. Нажмите "Publish" если еще не опубликовали
5. Обновите страницу сайта (Ctrl+R)
```

### Проблема: Ссылки не работают

```
☑ Решение:
1. Проверьте, что Slug заполнен правильно
2. Проверьте, что страница существует (например /about)
3. Проверьте консоль браузера (F12) на ошибки
```

### Проблема: Неправильный порядок пунктов

```
☑ Решение:
1. Откройте документ в /studio
2. Отредактируйте поле "Order" для каждого пункта
3. Используйте числа: 1, 2, 3... или 10, 20, 30...
4. Нажмите "Publish"
5. Обновите страницу
```

## 💾 Сохранение и публикация

### Два способа работы:

#### 1️⃣ Автоматическое сохранение

- Sanity автоматически сохраняет черновики (Drafts)
- Вы видите текущее состояние формы

#### 2️⃣ Публикация

- Нажмите кнопку "Publish" для сохранения в production
- Меню на сайте обновится (может потребоваться обновить страницу)

**Совет**: Всегда нажимайте "Publish" после внесения изменений!

## 🎓 Полезные советы

1. **Сортировка**: Используйте Order 10, 20, 30 (вместо 1, 2, 3), так легче вставлять новые элементы
2. **Многоязычность**: Заполняйте все языки сразу, чтобы не забыть
3. **Иерархия**: Максимум 2 уровня (родитель + дети)
4. **Slug**: Проверьте, что страница существует по этому маршруту
5. **Тестирование**: Всегда проверяйте на мобильном после изменений

## 📞 Техническая справка

- **Документ ID**: `menuNavigation`
- **Тип**: Document
- **Можно ли удалить**: Нежелательно, но если необходимо, пересоздайте
- **Кеширование**: API результаты могут кешироваться (5 мин по умолчанию)
